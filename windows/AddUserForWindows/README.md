# 域用户/本地用户创建脚本

## 概述

该 PowerShell 脚本用于在 Windows 环境中快速创建用户账号。

脚本会自动检测当前计算机是否处于 Active Directory 域环境：

- 如果当前计算机已加入域，则创建域用户
- 如果当前计算机未加入域，则创建本地 Windows 用户

脚本支持手动指定用户名和密码，也支持自动生成随机用户信息。创建完成后，会在窗口中打印生成的用户信息，便于查看和记录。

## 功能特点

- 自动检测当前环境是域环境还是本地环境
- 域环境下自动使用当前域创建用户
- 非域环境下自动创建本地用户
- 启动时检查是否以管理员身份运行
- 支持指定用户名
- 支持指定密码或自动生成随机密码
- 支持一键生成随机用户
- 创建完成后打印用户信息
- 创建失败时显示错误原因并暂停窗口，方便排查问题
- 可用于任意 Active Directory 域环境

## 用户生成方式

脚本启动后会提供两个选项。

### 1. 创建指定用户名和密码的用户

用户可以自行输入用户名。

输入用户名时不要包含 `@域名` 部分，只需要输入用户名本身。

随后脚本会询问是否指定密码：

- 如果选择指定密码，则由用户手动输入密码
- 如果不指定密码，则脚本自动生成随机密码

### 2. 创建随机用户

脚本会自动生成用户名和密码。

随机用户名规则：

```text
mu + 四位随机数字
```

示例：

```text
mu0427
mu8351
```

随机密码规则：

- 长度为 10 位
- 包含大写字母
- 包含小写字母
- 包含数字
- 不包含特殊字符

## 运行要求

### 权限要求

请以管理员身份运行 PowerShell。

如果当前 PowerShell 不是管理员权限，脚本会提示用户并退出。

### 域环境要求

如果需要创建域用户：

- 当前计算机需要加入 Active Directory 域
- 执行脚本的账号需要具备创建域用户的权限
- 当前计算机需要安装 Active Directory PowerShell 模块

通常可以通过安装 RSAT Active Directory 工具获得该模块。

### 本地环境要求

如果当前计算机未加入域，脚本会自动创建本地用户。

## 使用方法

以管理员身份打开 PowerShell，然后执行：

```powershell
powershell -ExecutionPolicy Bypass -File ".\Add-DomainOrLocalUser.ps1"
```

如果使用中文提示版本：

```powershell
powershell -ExecutionPolicy Bypass -File ".\Add-DomainOrLocalUser.zh-CN.ps1"
```

## 输出信息

用户创建成功后，脚本会打印以下信息：

- 创建目标环境
- 域名或计算机名
- 用户名
- 用户主体名称
- 密码

示例输出字段：

```text
Target
Domain
UserName
UserPrincipalName
Password
```

在本地环境中，`UserPrincipalName` 可能为空。

## 常见失败原因

如果用户创建失败，可能原因包括：

- PowerShell 未以管理员身份运行
- 指定的密码不符合域或本地密码策略
- 用户名已经存在
- 当前计算机未安装 ActiveDirectory PowerShell 模块
- 当前账号没有创建域用户的权限

脚本会在失败时显示错误信息，并暂停窗口，方便用户查看原因。

## 注意事项

- 创建域用户时，请确认当前账号具备足够权限
- 手动输入用户名时不要包含 `@域名`
- 手动指定密码时，需要符合当前域或本地系统的密码策略
- 随机生成的密码会直接显示在窗口中，请妥善保存
- 建议在受控环境中运行该脚本
